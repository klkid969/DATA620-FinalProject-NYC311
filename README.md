# DATA 620 Final Project: NYC 311 Complaint Analysis

This project analyzes NYC 311 complaint data to explore complaint patterns, response times, borough-level differences, and text-based insights from complaint descriptors.

## Project Overview

The goal of this project is to better understand how NYC 311 service requests are reported and handled. The analysis focuses on complaint status, response time, complaint type, borough distribution, and common words found in complaint descriptors.

## Dataset

The dataset used in this project comes from NYC Open Data:

[https://data.cityofnewyork.us/Social-Services/311-Service-Requests-from-2020-to-Present/erm2-nwe9/about_data](https://data.cityofnewyork.us/Social-Services/311-Service-Requests-from-2020-to-Present/erm2-nwe9/about_data)

A sample of the dataset was used to keep the analysis manageable and reproducible.

## Tools Used

- Python
- pandas
- matplotlib
- collections / Counter
- regular expressions
- Google Colab

## Key Areas of Analysis

- Complaint status distribution
- Response time calculations
- Fastest average response times by complaint type
- Common words in complaint descriptors
- Complaint patterns by borough

## Key Takeaway

This project showed that NYC 311 complaint data can reveal useful patterns about how public service requests are reported and processed. The analysis also raised an important distinction between a complaint being marked as closed and the issue being fully resolved from a resident’s perspective.
